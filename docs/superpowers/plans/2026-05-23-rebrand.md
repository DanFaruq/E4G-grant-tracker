# App Rebrand Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Modernize the E4G Grant Tracker with a premium design system — richer typography scale, improved spacing/shadows, semantic color tokens, and Framer Motion page/component animations — without breaking any existing functionality.

**Architecture:** All visual changes are additive. The CSS custom properties in `globals.css` are extended with new semantic tokens (surface layers, elevated shadow values, improved spacing scale). A new `components/ui/page-transition.tsx` client component wraps dashboard pages in Framer Motion. Sidebar and grant components receive subtle animation polish. No existing props, routes, or server actions are changed.

**Tech Stack:** Tailwind CSS v4 (OKLch color system already in place), Framer Motion (already installed), shadcn/ui, CSS custom properties.

**Execute last:** Run this plan only after Stakeholder Management and Team Activity Tracker are fully working, so new visual polish applies to all three features at once.

---

## Design System Decisions (from UI/UX Pro Max analysis)

Based on the existing E4G brand (navy + burnt orange, Plus Jakarta Sans) and the premium SaaS product type, the following changes are scoped:

| Area | Current | After Rebrand |
|------|---------|---------------|
| Typography scale | No explicit scale; relies on Tailwind defaults | Add `--text-xs` through `--text-4xl` with explicit line-heights |
| Card surfaces | Flat white / single card token | Add `--card-elevated`, `--card-sunken` surface layers |
| Shadow scale | No custom shadow tokens | Add `--shadow-xs` through `--shadow-xl` |
| Page transitions | CSS `animate-fade-up` utility only | Framer Motion `AnimatePresence` + `motion.div` wrapper |
| Component hover states | Basic `hover:shadow-md` | Micro-animation on key interactive cards |
| Sidebar | Static nav links | Framer Motion indicator slide animation |
| Dashboard cards | Static | Staggered entrance animations |

- [ ] **Step 2.1: Create PageTransition component**

```typescript
// components/ui/page-transition.tsx
"use client"

import { motion, AnimatePresence } from "framer-motion"
import { usePathname } from "next/navigation"

const variants = {
  hidden:  { opacity: 0, y: 8 },
  visible: { opacity: 1, y: 0 },
  exit:    { opacity: 0, y: -4 },
}

interface PageTransitionProps {
  children: React.ReactNode
}

export function PageTransition({ children }: PageTransitionProps) {
  const pathname = usePathname()

  return (
    <AnimatePresence mode="wait">
      <motion.div
        key={pathname}
        variants={variants}
        initial="hidden"
        animate="visible"
        exit="exit"
        transition={{ duration: 0.18, ease: [0.4, 0, 0.2, 1] }}
        className="flex-1 min-h-0 overflow-y-auto"
      >
        {children}
      </motion.div>
    </AnimatePresence>
  )
}
```
